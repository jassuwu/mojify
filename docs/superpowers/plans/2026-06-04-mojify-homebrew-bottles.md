# Mojify Homebrew Bottles Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add true Homebrew bottles for `brew install jassuwu/tap/mojify` while keeping the current source-building formula and fallback behavior.

**Architecture:** Keep `jassuwu/mojify` tag pushes as the human release gate. The source repo release workflow publishes existing GitHub Release tarballs, builds Homebrew bottles on native GitHub-hosted runners, uploads bottle artifacts to `jassuwu/homebrew-tap` GitHub Releases, merges generated bottle metadata into the tap formula, and pushes the tap update. This intentionally avoids converting Mojify to a binary formula or cask.

**Tech Stack:** Go, Bun/Turbo, GitHub Actions, Homebrew formula DSL, `brew install --build-bottle`, `brew bottle --json`, `brew bottle --merge`, `gh release`, `ffmpeg` and `yt-dlp` as external runtime dependencies.

---

## Decisions

- Homebrew remains a formula installed as `jassuwu/tap/mojify`.
- Bottles are generated by Homebrew, not reused from Mojify's existing release tarballs.
- Mojify release tarballs stay in `jassuwu/mojify` GitHub Releases for non-Homebrew users.
- Homebrew bottle artifacts live in `jassuwu/homebrew-tap` GitHub Releases.
- The first required bottle matrix is macOS Apple Silicon, macOS Intel, and Linux x86_64.
- Linux arm64 Homebrew bottles are deferred until there is a boring native hosted-runner path; Linux arm64 users keep source fallback.
- The release remains fully automatic after the maintainer pushes a valid `vYYYY.MM.DD.BUILD` tag.
- `TAP_GITHUB_TOKEN` continues to be the only cross-repo write token, scoped to `jassuwu/homebrew-tap` contents.

## File Structure

- `.github/workflows/release.yml`: replace the single Ubuntu release job with a prepare job, a native bottle matrix job, and a tap publishing job.
- `docs/adr/0028-add-homebrew-bottles-with-source-fallback.md`: create the architectural decision record for true bottles plus source fallback.
- `docs/release.md`: update the public runbook to document bottle publishing, tap bottle release assets, source fallback, and bottle smoke tests.
- `README.md`: update the Homebrew install sentence so it no longer says Homebrew always builds Mojify from source.
- `scripts/render-homebrew-formula.sh`: add explicit formula version metadata so calendar build tags do not depend on Homebrew URL version inference.
- `jassuwu/homebrew-tap/Formula/mojify.rb`: generated and pushed by the release workflow; do not edit it manually in this source repo plan.

## Task 1: Record the Bottle Distribution Decision

**Files:**
- Create: `docs/adr/0028-add-homebrew-bottles-with-source-fallback.md`

- [ ] **Step 1: Create the ADR**

Create `docs/adr/0028-add-homebrew-bottles-with-source-fallback.md`:

```markdown
# Add Homebrew bottles with source-build fallback

Mojify will keep Homebrew distribution as a real formula built from tagged source, and will add Homebrew-generated bottles for supported hosts. This means `brew install jassuwu/tap/mojify` can pour a prebuilt Homebrew keg when a matching bottle exists, while unsupported hosts and explicit `--build-from-source` installs still use the formula's `go build` path.

The bottle artifacts are separate from the normal GitHub Release tarballs. Mojify's source repository continues to publish macOS and Linux tarballs for direct download, while `jassuwu/homebrew-tap` GitHub Releases store Homebrew bottle tarballs generated by `brew bottle`. The formula keeps `ffmpeg` and `yt-dlp` as runtime dependencies, and keeps `go` as a build-only dependency for source fallback and bottle generation.

The release tag remains the human publishing gate. A valid `vYYYY.MM.DD.BUILD` tag runs the release workflow, publishes the existing source-repo release artifacts, builds bottles on native Homebrew runners, uploads bottle artifacts to the tap repository, merges the generated `bottle do` block into `Formula/mojify.rb`, and pushes the tap update with `TAP_GITHUB_TOKEN`.

The first bottle set is macOS Apple Silicon, macOS Intel, and Linux x86_64. Linux arm64 bottles are deferred until a native hosted-runner path is straightforward; Linux arm64 keeps source-build fallback. Signing, notarization, and cosign-style provenance remain deferred because Homebrew bottles are checksum-pinned formula artifacts rather than casks or app bundles.
```

## Task 2: Split Release Workflow into Prepare, Bottle, and Tap Publish Jobs

**Files:**
- Modify: `.github/workflows/release.yml`

- [ ] **Step 1: Replace `.github/workflows/release.yml` with a tag-driven bottle workflow**

Replace `.github/workflows/release.yml` with:

```yaml
name: Release

on:
  push:
    tags:
      - "v*.*.*.*"

permissions:
  contents: write

env:
  FORMULA_NAME: mojify
  TAP_NAME: jassuwu/tap
  TAP_REPOSITORY: jassuwu/homebrew-tap

jobs:
  prepare:
    name: Publish CLI release and render formula
    runs-on: ubuntu-latest
    timeout-minutes: 45
    outputs:
      tag: ${{ steps.meta.outputs.tag }}
      version: ${{ steps.meta.outputs.version }}
      source_sha256: ${{ steps.source.outputs.sha256 }}
      bottle_release: ${{ steps.meta.outputs.bottle_release }}
      bottle_root_url: ${{ steps.meta.outputs.bottle_root_url }}
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Set up Go
        uses: actions/setup-go@v5
        with:
          go-version-file: go.mod

      - name: Capture release metadata
        id: meta
        shell: bash
        run: |
          set -euo pipefail
          tag="$GITHUB_REF_NAME"
          if [[ ! "$tag" =~ ^v[0-9]{4}\.[0-9]{2}\.[0-9]{2}\.[0-9]+$ ]]; then
            echo "Invalid Mojify release tag: $tag" >&2
            echo "Expected vYYYY.MM.DD.BUILD, for example v2026.06.02.145" >&2
            exit 1
          fi
          version="${tag#v}"
          bottle_release="mojify-${version}"
          bottle_root_url="https://github.com/${TAP_REPOSITORY}/releases/download/${bottle_release}"
          {
            echo "tag=${tag}"
            echo "version=${version}"
            echo "bottle_release=${bottle_release}"
            echo "bottle_root_url=${bottle_root_url}"
          } >> "$GITHUB_OUTPUT"

      - name: Install FFmpeg
        run: sudo apt-get update && sudo apt-get install -y ffmpeg

      - name: Test
        run: go test ./...

      - name: Package release artifacts
        shell: bash
        run: |
          set -euo pipefail
          scripts/package-release.sh "${{ steps.meta.outputs.version }}" dist/release

      - name: Version smoke
        shell: bash
        run: |
          set -euo pipefail
          version="${{ steps.meta.outputs.version }}"
          tmp_dir="$(mktemp -d)"
          tar -xzf "dist/release/mojify_${version}_linux_amd64.tar.gz" -C "$tmp_dir"
          actual="$("$tmp_dir/mojify" --version)"
          rm -rf "$tmp_dir"
          expected="mojify ${version}"
          if [[ "$actual" != "$expected" ]]; then
            echo "Unexpected version output" >&2
            echo "Expected: $expected" >&2
            echo "Actual:   $actual" >&2
            exit 1
          fi

      - name: Publish GitHub Release
        shell: bash
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        run: |
          set -euo pipefail
          version="${{ steps.meta.outputs.version }}"
          if gh release view "$GITHUB_REF_NAME" --repo "$GITHUB_REPOSITORY" >/dev/null 2>&1; then
            gh release upload "$GITHUB_REF_NAME" dist/release/* \
              --repo "$GITHUB_REPOSITORY" \
              --clobber
            gh release edit "$GITHUB_REF_NAME" \
              --repo "$GITHUB_REPOSITORY" \
              --title "Mojify ${version}"
          else
            gh release create "$GITHUB_REF_NAME" dist/release/* \
              --repo "$GITHUB_REPOSITORY" \
              --title "Mojify ${version}" \
              --generate-notes
          fi

      - name: Compute source archive checksum
        id: source
        shell: bash
        run: |
          set -euo pipefail
          archive_url="https://github.com/${GITHUB_REPOSITORY}/archive/refs/tags/${GITHUB_REF_NAME}.tar.gz"
          archive_path="$(mktemp)"
          curl -fsSL "$archive_url" -o "$archive_path"
          sha256="$(shasum -a 256 "$archive_path" | awk '{print $1}')"
          echo "sha256=$sha256" >> "$GITHUB_OUTPUT"

      - name: Render Homebrew formula
        shell: bash
        run: |
          set -euo pipefail
          mkdir -p dist/homebrew/Formula
          scripts/render-homebrew-formula.sh "$GITHUB_REF_NAME" "${{ steps.source.outputs.sha256 }}" > dist/homebrew/Formula/mojify.rb
          ruby -c dist/homebrew/Formula/mojify.rb

      - name: Upload rendered Homebrew formula
        uses: actions/upload-artifact@v4
        with:
          name: homebrew-formula
          path: dist/homebrew/Formula/mojify.rb
          if-no-files-found: error

  bottle:
    name: Build Homebrew bottle (${{ matrix.artifact_name }})
    needs: prepare
    runs-on: ${{ matrix.os }}
    timeout-minutes: 75
    strategy:
      fail-fast: false
      matrix:
        include:
          - os: macos-26
            artifact_name: macos-arm64
          - os: macos-15-intel
            artifact_name: macos-intel
          - os: ubuntu-latest
            artifact_name: linux-x86_64
    steps:
      - name: Checkout Homebrew tap
        uses: actions/checkout@v4
        with:
          repository: ${{ env.TAP_REPOSITORY }}
          token: ${{ secrets.TAP_GITHUB_TOKEN }}
          path: homebrew-tap

      - name: Download rendered Homebrew formula
        uses: actions/download-artifact@v4
        with:
          name: homebrew-formula
          path: rendered-formula

      - name: Install rendered formula into tap checkout
        shell: bash
        run: |
          set -euo pipefail
          mkdir -p homebrew-tap/Formula
          cp rendered-formula/mojify.rb homebrew-tap/Formula/mojify.rb
          ruby -c homebrew-tap/Formula/mojify.rb

      - name: Set up Homebrew
        uses: Homebrew/actions/setup-homebrew@main

      - name: Build and bottle formula
        shell: bash
        env:
          HOMEBREW_NO_INSTALL_CLEANUP: "1"
          HOMEBREW_NO_INSTALLED_DEPENDENTS_CHECK: "1"
        run: |
          set -euo pipefail
          brew tap "$TAP_NAME" "$PWD/homebrew-tap"
          brew audit --formula "$TAP_NAME/$FORMULA_NAME"
          brew install --build-bottle "$TAP_NAME/$FORMULA_NAME"
          brew test "$TAP_NAME/$FORMULA_NAME"
          brew bottle --json --root-url "${{ needs.prepare.outputs.bottle_root_url }}" "$TAP_NAME/$FORMULA_NAME"
          shopt -s nullglob
          bottle_tarballs=( *.bottle*.tar.gz )
          bottle_json=( *.bottle.json )
          if (( ${#bottle_tarballs[@]} == 0 )); then
            echo "brew bottle did not produce bottle tarballs" >&2
            exit 1
          fi
          if (( ${#bottle_json[@]} == 0 )); then
            echo "brew bottle did not produce bottle JSON metadata" >&2
            exit 1
          fi
          printf '%s\n' "${bottle_tarballs[@]}" "${bottle_json[@]}"

      - name: Upload bottle artifacts
        uses: actions/upload-artifact@v4
        with:
          name: bottle-${{ matrix.artifact_name }}
          path: |
            *.bottle*.tar.gz
            *.bottle.json
          if-no-files-found: error

  publish-tap:
    name: Publish Homebrew tap bottles
    needs:
      - prepare
      - bottle
    runs-on: ubuntu-latest
    timeout-minutes: 30
    steps:
      - name: Checkout Homebrew tap
        uses: actions/checkout@v4
        with:
          repository: ${{ env.TAP_REPOSITORY }}
          token: ${{ secrets.TAP_GITHUB_TOKEN }}
          path: homebrew-tap

      - name: Download rendered Homebrew formula
        uses: actions/download-artifact@v4
        with:
          name: homebrew-formula
          path: rendered-formula

      - name: Download bottle artifacts
        uses: actions/download-artifact@v4
        with:
          pattern: bottle-*
          path: bottle-artifacts
          merge-multiple: true

      - name: Install rendered formula into tap checkout
        shell: bash
        run: |
          set -euo pipefail
          mkdir -p homebrew-tap/Formula
          cp rendered-formula/mojify.rb homebrew-tap/Formula/mojify.rb
          ruby -c homebrew-tap/Formula/mojify.rb

      - name: Set up Homebrew
        uses: Homebrew/actions/setup-homebrew@main

      - name: Publish tap bottle release
        shell: bash
        env:
          GH_TOKEN: ${{ secrets.TAP_GITHUB_TOKEN }}
        run: |
          set -euo pipefail
          shopt -s nullglob
          bottles=( bottle-artifacts/*.bottle*.tar.gz )
          if (( ${#bottles[@]} == 0 )); then
            echo "No bottle tarballs downloaded" >&2
            exit 1
          fi
          release="${{ needs.prepare.outputs.bottle_release }}"
          if gh release view "$release" --repo "$TAP_REPOSITORY" >/dev/null 2>&1; then
            gh release upload "$release" "${bottles[@]}" \
              --repo "$TAP_REPOSITORY" \
              --clobber
            gh release edit "$release" \
              --repo "$TAP_REPOSITORY" \
              --title "Mojify Homebrew bottles ${{ needs.prepare.outputs.version }}"
          else
            gh release create "$release" "${bottles[@]}" \
              --repo "$TAP_REPOSITORY" \
              --title "Mojify Homebrew bottles ${{ needs.prepare.outputs.version }}" \
              --notes "Homebrew bottles for Mojify ${{ needs.prepare.outputs.version }}."
          fi

      - name: Merge bottle metadata into tap formula
        shell: bash
        run: |
          set -euo pipefail
          shopt -s nullglob
          json_files=( bottle-artifacts/*.bottle.json )
          if (( ${#json_files[@]} == 0 )); then
            echo "No bottle JSON files downloaded" >&2
            exit 1
          fi
          cd homebrew-tap
          brew tap "$TAP_NAME" "$PWD"
          json_args=()
          for json in "${json_files[@]}"; do
            json_args+=( "../$json" )
          done
          brew bottle --merge --write --no-commit "${json_args[@]}"
          ruby -c Formula/mojify.rb
          brew audit --formula "$TAP_NAME/$FORMULA_NAME"
          git diff -- Formula/mojify.rb

      - name: Push Homebrew tap update
        shell: bash
        run: |
          set -euo pipefail
          cd homebrew-tap
          git config user.name "github-actions[bot]"
          git config user.email "41898282+github-actions[bot]@users.noreply.github.com"
          git add Formula/mojify.rb
          if git diff --cached --quiet; then
            echo "Homebrew formula already up to date."
            exit 0
          fi
          git commit -m "mojify ${{ needs.prepare.outputs.version }}"
          git push
```
