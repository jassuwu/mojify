# Start installable CLI distribution with GitHub Releases and Homebrew

Mojify will make its first installable CLI distribution stage native-script-driven rather than GoReleaser-driven: stable calendar + build tags in the shape `vYYYY.MM.DD.BUILD` will publish prebuilt macOS and Linux CLI tarballs plus checksums through GitHub Releases, while `brew install jassuwu/tap/mojify` will use a real source-building Homebrew formula in `jassuwu/homebrew-tap`. GitHub Actions and checked-in scripts own release packaging, formula rendering, version injection, checksum generation, release publishing, and tap updates; GoReleaser is not used for this stage because its Homebrew path is oriented around installing prebuilt upstream artifacts and is moving toward casks, while Mojify wants CLI formula UX with Homebrew building from a tagged source archive. The first binary matrix is `darwin/amd64`, `darwin/arm64`, `linux/amd64`, and `linux/arm64`; `mojify --version` and `mojify version` should print a single user-facing line like `mojify 2026.06.02.145`. Real publishing is tag-only on GitHub-hosted runners; local snapshot QA and normal CI should exercise the same packaging script without publishing or touching the tap. GitHub Releases should publish tarballs and checksums first, then the workflow should hash the GitHub tag source archive, render `Formula/mojify.rb`, and push that formula to the tap with a dedicated `TAP_GITHUB_TOKEN`; if the tap update fails after release publishing, the workflow should fail loudly without automatically deleting or retagging the GitHub Release. Creating the tap is a manual operator setup step, not an automated side effect of the Mojify release workflow, and the release runbook should remain public-safe by documenting repository names, secret names, token purpose, and least-privilege scope without token values or private account material. Windows support is WSL-only for this stage, npm wrappers, desktop apps, landing-site deployment, Linux distro packages, native Windows binaries, `go install` as an advertised install path, signing, notarization, shell completions, manpages, bottles, custom source tarballs, and cosign-style provenance are deferred; Homebrew installs declare `ffmpeg` and `yt-dlp` as formula dependencies, tarball installs document runtime tools as prerequisites, and missing-tool handling stays as command-specific runtime dependency hints rather than a new `doctor` command.
